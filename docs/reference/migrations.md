# Migration Log

All migration files live in `packages/core/src/storage/migrations/`. They are applied in sorted filename order by `openDb()` via `runMigrations()`. Each migration is recorded in the `_migrations` table (applied_at timestamp) and skipped on subsequent runs (idempotent).

## Notes

- Files prefixed with a number (001–014) are legacy sequential migrations.
- Files prefixed with a Unix timestamp follow the current convention: `<unix-timestamp>_<verb>_<noun>.sql`.
- BREAKING changes (embedding dim changes, table drops) must be labeled **BREAKING** in this log and documented in `docs/reference/schema.md`.
- After any migration, run `pnpm -r typecheck` and `pnpm --filter @lifecoach/core test --run`.
