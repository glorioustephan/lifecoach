# Embedding Eval — /embedding-eval

Run embedding evaluation: **$1**

## Step 1 — Locate eval definition

Read `docs/evals/$1.md`. If it does not exist, stop:

```
Error: No eval definition found at docs/evals/$1.md

To create one, write a file at that path with frontmatter:
  title, slug, owner, description, query_set, expected_results, scoring_method
Then re-run this skill.
```

Parse the eval definition for:
- `query_set` — path to the JSONL or JSON file with queries
- `expected_results` — path to ground-truth file (if any)
- `scoring_method` — e.g. `recall@5`, `ndcg@10`, `mrr`
- `script` (optional) — explicit script path override

## Step 2 — Locate eval script

Check for an eval script in this order:
1. `script:` field in the eval frontmatter (explicit override)
2. `packages/core/src/embeddings/eval/$1.ts`
3. `packages/core/src/embeddings/eval/$1.js`
4. `packages/core/src/embeddings/eval/run-eval.ts` (generic runner)

If no script is found:
```
Warning: No eval script found for "$1". Reporting eval definition only.
Implement a script at packages/core/src/embeddings/eval/$1.ts to enable automated runs.
```
Skip to step 4 (write a result stub noting the missing script).

## Step 3 — Run eval script

Run the eval using pnpm:

```bash
pnpm --filter @lifecoach/core tsx <script-path> --eval $1 --output /tmp/eval-$1-result.json
```

Wait for completion. Capture stdout and stderr.

If the script exits non-zero, include the error output in the results file and mark
`status: error` in the frontmatter.

## Step 4 — Compute output path

```bash
date +%Y-%m-%d
```

Output path: `docs/evals/results/<date>-$1.md`

## Step 5 — Write results file

Write `docs/evals/results/<date>-$1.md`:

```markdown
<!-- GENERATED by embedding-eval skill on <date>. Do not edit by hand. Re-run via /embedding-eval $1. -->
---
title: Eval Results — $1 — <date>
slug: <date>-$1
audience: [agents]
owner: voyage-embeddings-engineer
status: <done|error>
created: <date>
updated: <date>
source: generated
generator: embedding-eval
eval_definition: docs/evals/$1.md
---

## Summary

- Eval set: $1
- Run date: <date>
- Script: <script-path or "not found">
- Status: <done|error>
- Scoring method: <scoring_method from definition>

## Results

<!-- If script ran successfully, paste or summarize the JSON output here -->
<results or "Script not found — manual run required">

## Errors / warnings

<stderr output if any, or "none">

## Recommendations

<!-- voyage-embeddings-engineer: interpret these results and suggest next steps -->
```

## Step 6 — Print confirmation

```
Results written: docs/evals/results/<date>-$1.md
Eval definition: docs/evals/$1.md
Status: <done|error>

Next: voyage-embeddings-engineer should review results and update the eval definition
if thresholds need adjustment.
```
