```markdown
# lifecoach — Contributor Snapshot

| @lifecoach/mcp-alpaca-server | packages/mcp-alpaca-server/ | Alpaca investment tools |

## Key commands

| Command | Purpose |
|---------|---------|
| `/council [topic]` | Fan out to all 3 advisors; get cross-domain brief + synthesis |
| `/handoff [brief-path]` | Translate a ready brief into engineer-actionable tasks |
| `/brief-status [domain?]` | Table of all briefs by status |
| `/arch-snapshot` | This document |
| `/domain-brief [domain] [slug] [summary]` | Create a new advisory brief |
| `/new-plugin [name] [domain]` | Scaffold brief + engineer spec pair |
| `/new-migration [short-name]` | Scaffold a SQL migration stub |
| `/mcp-tool-spec [tool-name]` | Write MCP tool engineer spec |
| `/doc-refresh [scope?]` | Audit docs/ for stale pages |
| `/embedding-eval [eval-set]` | Run embedding evaluation and save results |

## Entry points

- **Dev:** `pnpm dev` (from root or packages/web)
- **API:** `pnpm --filter @lifecoach/core start`
- **Type check:** `pnpm -r typecheck`
- **Background jobs:** `pm2 start ecosystem.config.js`

## Where to start

1. Read `CLAUDE.md` → `AGENTS.md` → `docs/index.md` (if it exists)
2. Run `/brief-status` to see what domain advisors have already specified
3. Pick your agent role from the council table above
4. Check `docs/briefs/` for `status: ready` briefs awaiting your domain
```
